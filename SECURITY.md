# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please **DO NOT** open a public issue.

### How to Report:
1. **Email me directly** at: sanjayksk1712@gmail.com
2. **Include:**
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Your suggested fix (if any)

### Response Timeline:
- Initial acknowledgment: Within 24-48 hours
- Investigation: 3-5 days
- Fix/Update: As soon as possible

## Security Considerations

### This is a Learning Repository

⚠️ **Important Notes:**
- This repo contains educational materials and practice code
- Code here is for learning purposes, NOT production use
- Database credentials and sensitive data are NOT included
- Do NOT use real passwords or API keys in this repo

### Best Practices:
✅ Never commit sensitive information (passwords, API keys, tokens)
✅ Review code before using in production
✅ Update your database credentials regularly
✅ Keep MySQL server updated
✅ Use proper authentication in real applications

### What's NOT Covered:
- This is a student learning repo
- Not for commercial use
- No guaranteed support
- Educational code quality standards only
